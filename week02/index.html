Week 02 Notes & Exercises

Notes

  • I wasn't sure what the difference between const, let, and var was in js, so I looked it up. When you make a var within a function, it can only be called in that function. However, if you declare it outside of a function, it can be used anywhere. Var can also be redefined at any point and not cause an error.

    Let is the preferred declaration for variables. It is block scoped, which means you can redefine it in a different block sucessfully, but if you call it outside of that redefined block, it will log the original declared value. In a sense, two lets labled the same in different blocks are considered different variables.

    Const, like let, can only be called within its own block, and they can't ever be redefined or re-declared. However, if you declare a const with code such as const d = value: 4, you can later change the value by going d.value = 8.

  • Local scope vs global scope: Global scope variables are declared outside of a block and can be used anywhere in the code. Local scope variables are declared within a block and usually can only be used within that block. Variables defined without let or const are automatically global scope.
  • You can use backslashes (\) to avoid the function associated with certain symbols. For example, if you had string 'It's fun', your single quotes would only identify 'It' as the string. However, if you added this backslash: 'It\'s fun', it automatically skips over that quote mark. If you actually want to write a backslash, you'll need to backslash it with an additional backslash! ex: "This is the backslash: \\".
  • There are some cool things you can do with strings, like discover the length (name.length), change everything name.toUpperCase or name.toLowerCase, or discover which character is at which point (name.charAt(1)). REMEMBER that there is a position 0 in javascript. Position 1 is the second character. There are plenty other string methods.
  • I learned that through template literals, you can use the backtick character (`) instead of the quote mark to note strings. This makes it easier to add quotation marks in your strings. It also eliminates the need for + signs between strings and variables you want to insert into your code. You use curly brackets and $ instead, ex: `Hello ${name}!`
  • I also discovered the difference between parseInt and parseFloat. ParseInt will convert a string into an integer, and return NaN if there is no number. ParseFloat will convert a string into a "floating" number, which is usually a decimal.
  • Some useful number methods are toFixed(), where you can round something to (x) number of decimals; and toPrecision(), which rounds a number to a significant amount of figures (so large numbers go to exponential notation, etc).
  • I didn't know you could put differnt data types into an array. For example, you could have a number, string, and undefined data all in the same array.
  • I also learned some new methods you can use with arrays, like arrayName.pop(), which displays the last item in the array and then removes it from the array. Or arrayName.shift() which does the same thing to the first item in the array. There's also .push('new item') and .unshift('new item'), which add a new item to the end and beginning of the array, respectively. There's also .join() that turns all of the array items into one long string.
  • Use a set if you don't want values to be repeated. To make a set: let firstTry = new Set(). Then do firstTry.add(1) to add the number 1 to the set. Duplicate adds will be ignored, and strings will be seperated into letters and duplicate letters will be deleted. ex: let testSet = new Set('hello'). testSet = {'h', 'e', 'l', 'o'}. However, if you add each word seperately, it keeps the string together. ex: let testSet = new Set.add('hello').add('neighbor'). Note that sets don't have index numbers and they read strings and numbers, etc differently. You can convert a set into an array by doing this: let testSet = [... testArray].
