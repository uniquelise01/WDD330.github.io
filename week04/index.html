Notes

  • When creating forms, the most common types of form tags are <input>, (which can be text, password, radio buttons, etc) <select>, (which is for drop-down lists) <textarea>, and <button>.
  • Most of what I learned this week was how to apply principles in old chapters to things I was more familiar with, like forms and common html.
  • There are three characteristics of object-oriented programing: Encapsulation means only the essential functionalities are shows to the user, and all the methods etc are hidden inside the object programming. Polymorphism is the ability to mix up what goes into the object programming with the same resultâ€”or the same process can be used for different objects. And lastly, OOPs must have inheritance, meaning it can be updated while keeping the same previous functionality.
  • You can use the get() and set() methods to control how a property is set and what value is returned. So you can redefine a value both ways.
  • You can use the mixin() function to combine mutliple arrays, objects, functions, etc, without creating an inheritance chain.
  • The value of this doesn't stay consistent if you nest functions, so you can solve that by using that = this and/or bind(this) as a method in your functions.
  • Modular javascript is something like creating seperate js files for specific purposes (ex: you create a file for all Ajax projects and reference it when needed). You can import one modular js into another js file. You would do so like this: import {functionName, otherFunctionName} from './stats.js'; although there are several ways you can import different types of js.
