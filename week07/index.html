Week 07 Notes & Exercises

Notes

  • I didn't know you could use methods on functions. For example, you can find a function.length or function.call(). The .apply method works like the .call() method, except your arguments must be provided as an array.
  • Apparently you can also create your own properties to functions, like square.description = "blah blah blah."
  • You can create an immediately invoked function expression (or IIFE, "iffy") that is anonymous but is invoked as soon as it is defined and only runs once. You do this by putting the function in parentheses, and then putting () parentheses after that. You don't have to give the function a name because of that. Any variables declared in the IIFE will disappear after the function has been invoked.
  • You can also re-define functions within the function, so it runs code a certain way the first time, and then the second time it runs the code differently.
  • Recursive functions are those that keep calling themselves until a certain criteria is met. There's an example of this in my Practice link js.
  • I don't feel like I understand promises well enough to explain them in my notes, but I have a basic understanding that it's some sort of code that is set up similar to a class, except you need a resolve() and reject() function depending on whether or not the promise operation in successful.
  • Apparently there is such a thing as "pure" and "impure" functions. Explained simply, pure function don't change the value of a pre-established variable, don't have side-effects, always have arguments, and always return the same result when given the same arguments.
  • You can add a callback function as an argument in another function that will run as soon as the first function is complete. For example, function findHypotenuse(2, 3, square) would run the findHypotenuse function first, and then run the returning value through the square function.
  • Ajax allows you to communicate with a server and update your pages without reloading.It works asynchronously and with JSON.
  • APIs are "application programming interface" that grants you external access to another program or service. You can gain access to APIs using the fetch() method.
